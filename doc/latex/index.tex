\hypertarget{index_un}{}\doxysection{Introduction}\label{index_un}
Ce programme C est une reproduction du jeu de la vie.~\newline
 Le programme ne fonctionne que dans un terminal, et nécessite d\textquotesingle{}utiliser des fichiers textes pour le configurer.\hypertarget{index_deux}{}\doxysection{Structure de fichier}\label{index_deux}
Le projet comporte plusieurs répertoires \+: un répertoire source, un répertoire pour l\textquotesingle{}exécutable et un répertoire pour la documentation doxygen.~\newline
 Le répertoire source contient les fichiers sources du programme, ce sont ces fichiers qu\textquotesingle{}il faudra compiler si l\textquotesingle{}exécutable est supprimé.~\newline
 Par défaut, le répertoire bin contient déjà l\textquotesingle{}exécutable.~\newline
 \hypertarget{index_trois}{}\doxysection{Compilation du programme}\label{index_trois}
Si, pour une quelconque raison, vous devez recompiler le programme, voici un makefile simple qui vous le permettra \+: \begin{DoxyVerb}srcdir = ./src/
bindir = ./bin/
SRC = $(wildcard $(srcdir)*.c)
OBJ = $(subst $(srcdir),$(bindir),$(SRC:.c=.o))
PROG = $(bindir)exe
CC = gcc -Wall -g

all : $(PROG)
$(PROG) : $(OBJ)
    @echo 'Compilation de l executable'
    $(CC) $^ -o $@ -lm
./bin/%.o : ./src/%.c
    @echo 'Compilation des fichiers objets'
    $(CC) -c $^ -o $@ -lm\end{DoxyVerb}
 À noter que ce makefile doit se trouver dans le répertoire parent de src et bin.~\newline
 De même, faites bien attention à bien placer des tabulations là où elles sont nécessaires, makefile étant très sensible aux espaces. Si le message d\textquotesingle{}erreur \char`\"{}missing separator.
